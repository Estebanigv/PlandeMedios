```
├── .env.example              # Variables de entorno

```

## Estructura del Proyecto

```
plan-de-medios/
├── src/
│   ├── components/          # Componentes reutilizables
│   │   ├── features/       # Features (ContactForm, etc)
│   │   ├── layout/         # Layout components (Header, Footer)
│   │   ├── sections/       # Secciones de página (Hero, etc)
│   │   └── ui/             # Componentes UI básicos (Button, Input)
│   ├── content/            # Content collections (blog, servicios)
│   ├── data/               # Datos estáticos (site config, etc)
│   ├── layouts/            # Layouts de página
│   ├── lib/                # Utilidades y helpers
│   │   ├── validation.ts   # Validación Zod
│   │   ├── rate-limit.ts   # Rate limiting
│   │   └── email-service.ts # Email service
│   ├── pages/              # Páginas del sitio
│   │   ├── api/           # API endpoints
│   │   ├── blog/          # Blog
│   │   └── servicios/     # Servicios
│   └── styles/            # Estilos globales
├── public/                # Assets estáticos
└── dist/                  # Build output
```

## Convenciones

### Nomenclatura de Archivos
- Componentes: PascalCase (`ContactForm.astro`)
- Utilidades: camelCase (`validation.ts`)
- Páginas: kebab-case (`[slug].astro`)
- Estilos: kebab-case (`global.css`)

### Documentación
- Archivos principales: UPPER-KEBAB-CASE (`CONTACT-FORM-SETUP.md`)
- Archivos secundarios: kebab-case (`readme.md`)

### Código
- TypeScript para lógica compleja
- JSDoc para documentación de funciones
- Comentarios en español
- Props documentadas en componentes
